> # 💡 프로젝트 소감

### **1. CORS (Cross-Origin Resource Sharing) · CORB (Cross-Origin Read Blocking) ERROR**

- API 를 사용해 데이터를 받아 올 때, CORS ERROR 가 발생했는데 일반적으로 우리 쪽 서버에서 이를 무시할 (허용할) 사이트들을 설정해서 해당 에러를 해결하지만 이번 프로젝트에서는 서버를 사용할 수 없었기 때문에 일반적인 방법으로는 해결할 수 없었고 localhost 에 한하여 CORS 를 풀어주는 Chrome 확장 프로그램 (Allow CORS) 을 사용했습니다.

- 지도 API 를 선정하는 과정에서 카카오 맵 API 사용했을 때, CORB (Cross-Origin Read Blocking) 오류가 발생했고 이를 해결하기 위해 여러 방법을 찾아보다가 구글 지도 API 를 사용하여 CORB 오류를 해결했습니다.<br><br>


### **2. API 선정과 사용**

- API 를 사용해서 데이터를 받아오고 뿌리는 것에 익숙해지기 위한 연습을 해보는 것이 중점적인 목표였기에 주제를 먼저 선정하고 API 를 선정한 것이 아니라 API 를 먼저 선정하고 API 에 맞춰 웹 사이트를 개발했습니다.

- 위에서 말했듯이 API 사용에 익숙해지기 위해 하나의 API 만 사용하는 것이 아니라 최대한 다양한 API 를 사용하려 했고 공연예술통합전산망에서 3~4 개의 API, Google Maps API 를 사용했습니다. <br><br>


### **3. LocalStorage 를 사용한 정보 저장** 

- Spring, Spring Boot, Express.js 와 같은 Framework 에 대해 배우기 전이라 View Layer 에서 모든 요청을 처리해야 했기에 사용자가 메인 페이지에서 선택한 지역을 어떻게 넘겨줘야 할지 고민하다가 LocalStorage 를 활용했습니다.

- 메인 페이지에서 공연 정보 출력 페이지로 이동하는 경우, 공연 정보 출력 페이지에서 공연 정보 상세 조회 페이지로 이동하는 경우에서 LocalStorage 를 활용했으며 사용자 보안과 밀접하게 연관된 정보는 LocalStorage 에 저장하지 않고 API 호출과 관련된 정보만 LocalStorage 에 저장했습니다.
